TECHNOVA INC. — API AND BACKEND DEVELOPMENT STANDARDS
======================================================
Department: Engineering
Access: Employees and Above

REST API DESIGN STANDARDS
--------------------------
All APIs built at TechNova must follow these conventions for consistency and quality.

URL Conventions:
  Use lowercase letters and hyphens: /user-profiles (not /UserProfiles or /user_profiles)
  Use plural nouns for collections: /users, /models, /datasets
  Nested resources for relationships: /users/{id}/projects
  API versioning in URL: /v1/users, /v2/models
  Never use verbs in URLs: /users (not /getUsers or /fetchUsers)

HTTP Methods:
  GET    - Read data, never modifies anything
  POST   - Create a new resource
  PUT    - Replace an entire resource
  PATCH  - Partially update a resource
  DELETE - Remove a resource

Standard Response Codes:
  200 OK           - Successful GET, PUT, PATCH
  201 Created      - Successful POST (include Location header)
  204 No Content   - Successful DELETE
  400 Bad Request  - Invalid input or parameters
  401 Unauthorized - Missing or invalid authentication token
  403 Forbidden    - Authenticated but not authorized
  404 Not Found    - Resource does not exist
  422 Unprocessable - Validation errors
  429 Too Many Requests - Rate limit exceeded
  500 Internal Error - Server error (log and alert)

Standard JSON Response Format:
  Success:
    {
      "data": { ... },
      "meta": { "page": 1, "total": 100, "per_page": 20 }
    }

  Error:
    {
      "error": {
        "code": "VALIDATION_ERROR",
        "message": "Email address is not valid",
        "details": [{ "field": "email", "issue": "Invalid format" }]
      }
    }

AUTHENTICATION
--------------
All protected endpoints must require JWT Bearer token:
  Authorization: Bearer <token>

Token expiry: 1 hour for access tokens, 7 days for refresh tokens
JWT must contain: user_id, role, email, issued_at, expires_at
Never store tokens in localStorage — use sessionStorage or HttpOnly cookies
Token refresh must be handled silently in the client without forcing re-login

RATE LIMITING
-------------
Public endpoints: 30 requests per minute per IP
Authenticated endpoints: 100 requests per minute per user
Admin endpoints: 200 requests per minute per user
LLM endpoints: 10 requests per minute per user (due to cost)

Rate limit headers must be included in every response:
  X-RateLimit-Limit: 100
  X-RateLimit-Remaining: 87
  X-RateLimit-Reset: 1719800000 (Unix timestamp)

SECURITY REQUIREMENTS
---------------------
All endpoints must use HTTPS — HTTP requests must be redirected to HTTPS
No secrets in code — use AWS Secrets Manager or environment variables
SQL injection prevention: Always use parameterized queries, never string concatenation
Input validation: Validate and sanitize all user inputs using Pydantic models
CORS: Restrict allowed origins to known frontend domains (never use wildcard in production)
Logging: Never log passwords, tokens, credit card numbers, or PII

DATABASE STANDARDS
------------------
Primary database: PostgreSQL 15+
ORM: SQLAlchemy for Python services
Migrations: Alembic for all schema changes (never edit database directly in production)
Connection pooling: Use PgBouncer in production
Backups: Automated daily backups, retained for 30 days
Read replicas: Required for all read-heavy services
Indexes: Required on all foreign keys and frequently queried columns
Transactions: Use database transactions for all multi-step write operations

TESTING REQUIREMENTS
--------------------
Unit tests: Minimum 80 percent coverage (measured by pytest-cov)
Integration tests: Required for all API endpoints (use FastAPI TestClient)
Load testing: Required before launching any new public API (use Locust)
Test naming: test_should_<action>_when_<condition>

CI/CD PIPELINE
--------------
All PRs must pass:
  1. Linting (flake8 + black check)
  2. Type checking (mypy)
  3. Unit tests (pytest)
  4. Integration tests
  5. Security scan (bandit for Python)
  6. Docker image build

Deployment flow:
  Feature branch → PR → Code review (2 approvals) → Merge to main → Auto-deploy to staging → Manual promotion to production

DOCUMENTATION
-------------
All APIs must have OpenAPI documentation (auto-generated by FastAPI at /docs)
Keep docstrings updated — they appear in auto-generated docs
Postman collection must be maintained for all public APIs
Stored in: github.com/technova/api-collections

Contact: backend-platform@technova.com for backend infrastructure support.
